# AI MUSIC AGENT OZ - Architecture & Updates Log

이 문서는 시스템 설정 및 아키텍처 변경 사항을 지속적으로 분석하고 기억하기 위한 메모리 로그입니다. 향후 에이전트 간의 충돌을 방지하고 작업 이력을 바탕으로 시스템을 유지 보수하는 데 사용됩니다.

## [2026-04-07] PM2 스케줄러 분리 및 할당량 방어 로직 (429 Quota) 확인
### 🚨 문제 상황:
- **인스타그램/아린 에이전트(Ozpix_Instar_Arin)** 와 **AI MUSIC AGENT OZ** 가 개별 프로젝트임에도 불구하고 `ecosystem.config.js` 상에서 PM2 앱 이름(`OZ_MASTER_SCHEDULER`)을 공유하고 있었습니다.
- 이로 인해 PM2 레벨에서 프로세스가 서로 덮어씌워지는 심각한 **네이밍 충돌**이 발생했습니다.

### 🛠️ 해결 및 변경 내용:
1. **PM2 계정(이름) 분리**: `AI MUSIC AGENT OZ`의 스케줄러 앱 이름을 **`OZ_MUSIC_MASTER_SCHEDULER`** 로 확실하게 변경 및 분리.
2. **독립적 구동 확인**: 인스타그램 스케줄과 100% 독립적으로 음악 자율 생산 엔진(`run_autonomous.js`)이 8시간 간격으로 구동되도록 분리 재가동 (PM2 Started).

## [2026-04-07] 유튜브 자동 업로드 실패 원인 분석 및 저작권(Safety) 필터 통과 최적화
### 🚨 문제 상황:
- `OZ CAFE` 테마에서 컴필레이션을 생성할 때, 유튜브 업로드가 이뤄지지 않고 작업이 통째로 뻗어버리는 현상(`QUOTA_EXHAUSTED_ALL_KEYS` 에러)이 감지되었습니다.
- 로그 분석 결과, 빠른 재즈(홀수 번째 트랙) 생성을 위한 프롬프트 내부에 **"Miles Davis & John Coltrane vibes"** 라는 실존 아티스트 이름이 포함되어 있었습니다.
- AI 음악 생성 엔진(Lyria 등)들이 **저작권 필터(Safety Filter)** 로 이를 즉시 차단(Block)해 버렸고, 엔진이 이를 임시 네트워크 에러로 간주해 무한 재시도를 반복하다가 가용한 구글 API 할당량(Quota)을 전부 태워버린 것이 원인이었습니다.

### 🛠️ 해결 및 변경 내용:
1. `prompt_engineer.js` 내부 프롬프트 지시어에서 특정 실존 아티스트 이름을 제거하고, `1960s Fast Jazz style` 등의 보편적 스타일 묘사로 대체했습니다.
2. 테마명 변형에 따른 가사 탐색 스크립트의 데이터 불일치(Undefined) 버그도 함께 수정했습니다.
3. 이를 통해 생성 엔진이 저작권 차단에 막혀 할당량을 고갈시키는 악순환의 고리를 끊어냈습니다!

### ✅ 향후 방향성 기억 사항:
- 어떤 테마나 프롬프트를 AI 음악 엔진에 넘겨줄 때, 저작권 문제가 발생할 수 있는 **유명 가수/밴드 이름이나 곡 제목 묘사를 절대 포함하지 않도록 주의**해야 합니다. (Safety Block 유발)
- 실패 분석 후 문제의 원인이 단순 통신 트래픽 지연(429)인지, 내용 상의 차단(Safety 필터) 인지 정확히 구별해서 무의미한 API 재시도 낭비를 방지할 것.
